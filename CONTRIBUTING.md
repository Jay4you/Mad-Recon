# Contributing to Madrecon

Thank you for considering contributing to Madrecon.

## How to Contribute

### 1. Fork the repository
Create your own fork and clone it locally.

### 2. Create a new branch
Use a descriptive name, for example:
feature/add-nmap
fix/httpx-header-bug

### 3. Make your changes
Ensure your code:
- Follows Python best practices
- Handles exceptions safely
- Writes output files per tool
- Doesnâ€™t block the main execution flow

### 4. Test the tool
Run your version against a test domain.

### 5. Submit a pull request
Include:
- A clear title
- A meaningful description
- Reference to an issue if applicable

## Coding Style
- Use descriptive variable names
- Avoid unnecessary dependencies
- Keep each tool wrapper function simple
- Do not introduce emojis in CLI output or repo documentation

## Reporting Issues
Please include:
- Steps to reproduce
- Error output
- Expected behavior
- System details
